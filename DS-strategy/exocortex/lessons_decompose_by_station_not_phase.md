# Декомпозиция: станции с материальным выходом, не фазы планирования

**Контекст:** WP-3 (приземление 5 SEO-схем в реализацию). Исходная Ф1 формулировалась как «вход 5 схем → выход план», а между ними — 6 мыслительных операций в голове (рассмотреть → выбрать точку → расписать порядок → оценить → связать → найти узкие места). Пилот почувствовал разрыв.

**Урок (переносимый):**

1. **Резать по станциям, а не по фазам.** Если «фаза» содержит цепочку мыслительных шагов без промежуточных артефактов — это чёрный ящик, а не этап. Каждый шаг → станция с **материальным выходом** на бумагу (правило физичности I/O Декомпозитора, [[Процесс ≠ Артефакт-результат ≠ Артефакт-процесс]]).

2. **Единица декомпозиции = объект, проходящий станции** (схема → точка → плейбук → ресурс), а не «5 монолитных карточек». Объекты идут по станциям параллельно.

3. **Оценка ресурса — производное, не вход.** Нельзя требовать «дай объём/время/дропы» на входе: оценка появляется только ПОСЛЕ того, как расписан порядок операций (плейбук). Требовать выход раньше его входа = инверсия = тот самый разрыв.

4. **«Связать и найти узкие места» — отдельная финальная станция над всем набором.** Узкое место одного объекта в отрыве не находится; нужна станция сведения после того, как все объекты приземлены.

5. **Длинные операции → фоновые потоки, параллельно волнам, не отдельной волной.** Закупка/research/копирайт — непрерывные потоки со старта; если поставить их «волной 2», волна 2 будет ждать на старте. Поверх потоков — волны запуска по готовности ресурса.

6. **Экспертный выбор требует следа обоснования.** «Выбрать точку из экспертизы» — быстрый акт, но выход станции = выбор + 1-2 фразы почему. Без следа выбор станет чёрным ящиком, который не перепроверить позже (нужно для интерпретации минус-результата: «механика мёртвая» vs «точку выбрал неудачно»).

**Тест:** «у каждого мыслительного шага есть файл на выходе, и оценка стоит после своего входа?» Нет → чёрный ящик в декомпозиции.
